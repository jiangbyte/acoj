# FastAPI → Gin 完整移植计划

## 概述

将 `hei-fastapi` 的全部 Python 代码一字不差地翻译为 Go + Gin 实现。每个函数方法、内部实现细节、业务逻辑保持一致，仅适配 Go 语言特性和 Gin 框架。

---

## 一、核心设计决策

### 1.1 装饰器 → Gin 中间件映射

FastAPI 使用 Python 装饰器实现横切关注点（鉴权、日志、防重复），Go/Gin 没有装饰器，统一映射为 **路由级中间件**：

| FastAPI 装饰器 | Gin 等价实现                                      |
|---|-----------------------------------------------|
| `@HeiCheckLogin` | 路由级中间件 `auth.HeiCheckLogin()` — 检查Token是否有效（先查 Redis → 无数据直接返回 None → 有数据再 JWT 验签 → 返回数据。这样 Redis 无数据时跳过 JWT 解码开销） |
| `@HeiCheckPermission("sys:x:y")` | 路由级中间件 `auth.HeiCheckPermission([]string{"sys:x:y"})` |
| `@HeiCheckPermission(["a", "b"], mode=OR)` | 路由级中间件 `auth.HeiCheckPermission([]string{"a", "b"}, "OR")` |
| `@HeiCheckRole("admin")` | 路由级中间件 `auth.HeiCheckRole([]string{"admin"})` |
| `@HeiCheckRole(["a", "b"], mode=OR)` | 路由级中间件 `auth.HeiCheckRole([]string{"a", "b"}, "OR")` |
| `@HeiClientCheckLogin` | 路由级中间件 `auth.HeiClientCheckLogin()`           |
| `@HeiClientCheckPermission("x:y")` | 路由级中间件 `auth.HeiClientCheckPermission("x:y")` |
| `@HeiClientCheckRole("admin")` | 路由级中间件 `auth.HeiClientCheckRole("admin")`     |
| `@SysLog("操作名称")` | 路由级中间件 `log.SysLog("操作名称")`                   |
| `@NoRepeat(interval=3000)` | 路由级中间件 `auth.NoRepeat(3000)`                  |

**执行顺序保持一致**：外层装饰器 = 先注册的中间件，先执行。

```
Python: @SysLog → @HeiCheckPermission → @NoRepeat → handler
Go:     r.POST("/path", SysLog, HeiCheckPermission("..."), NoRepeat(3000), handler)
              先执行 ↑                                                         ↑ 后执行
```

### 1.2 权限元数据收集（替代反射）

FastAPI 通过 `endpoint._hei_permission = permission` 运行时反射收集路由权限。Go 不具备等价机制，采用 **编译期注册表**：

```go
// core/auth/permission_scan.go
var permissionRegistry []PermissionEntry

type PermissionEntry struct {
    Code   string // "sys:banner:page"
    Module string // "sys:banner"
    Name   string // "获取Banner分页"
}

func RegisterPermission(entry PermissionEntry) { ... }

// 在模块 init() 或 路由注册时调用：
RegisterPermission(PermissionEntry{
    Code: "sys:banner:page",
    Name: "获取Banner分页",
    Module: "sys:banner",
})
```

启动时调用 `RunPermissionScan()` 读取 `permissionRegistry`，按模块分组写入 Redis 缓存。

### 1.3 ContextVar → Gin Context

FastAPI 的 `ContextVar` 用于在请求范围内传递认证上下文。Go 中没有等价物，改用 `gin.Context` 的 `Set()` / `Get()`：

| Python | Go |
|---|---|
| `HeiAuthTool._request_var.set(request)` | `c.Set("hei_auth_tool", instance)` |
| `HeiAuthTool._request_var.get()` | `c.MustGet("hei_auth_tool")` |
| `HeiClientAuthTool._request_var.set(request)` | `c.Set("hei_client_auth_tool", instance)` |
| `HeiClientAuthTool._request_var.get()` | `c.MustGet("hei_client_auth_tool")` |
| `record.trace_id` logging filter | `c.GetString("trace_id")` 传给日志 |
| `get_trace_id()` | 从 `c` 读取或 `c.GetString("trace_id")` |

### 1.4 ORM 层

| Python (SQLAlchemy) | Go (Ent) |
|---|---|
| `Session = sessionmaker(bind=engine)` | `ent.Client` (包变量 `db.Client`) |
| `db = SessionLocal()` | `db.Client` |
| `db.execute(select(Model).where(...))` | `client.Model.Query().Where(...).All(ctx)` |
| `db.add(entity); db.commit()` | `client.Model.Create().SetXxx(...).Save(ctx)` |
| `db.scalars(query).all()` | `client.Model.Query().All(ctx)` |
| `get_db()` 依赖注入 | 从 `c.MustGet("ent_client")` 获取 |

### 1.5 异步 → 同步

FastAPI 大量使用 `async/await`。Go 中 Ent 和 go-redis 的 API 默认同步。

| Python (asyncio) | Go (同步) |
|---|---|
| `await redis_client.get(key)` | `redisClient.Get(ctx, key).Result()` |
| `await redis_client.setex(key, ttl, val)` | `redisClient.SetEx(ctx, key, val, ttl).Err()` |
| `await db.execute(...)` | `db.Query().All(ctx)` |
| `await asyncio.to_thread(...)` | 无需，Go 原生并发 |

### 1.6 路由分组规则

FastAPI 的 `AuthMiddleware` 通过 URL 路径正则判断鉴权方式。Gin 中保持完全一致的行为——**不依赖 Gin group 中间件做鉴权**，而是由全局 `AuthCheck` 中间件统一按路径匹配规则处理。Gin group 仅用于路由分组注册，不参与鉴权逻辑。

**路径鉴权规则**（与 Python 版 `AuthMiddleware.__call__` 完全一致）：

| 路径模式 | 鉴权方式 | 包含路由 |
|---|---|---|
| `GET /` | 无需鉴权（不匹配任何API模式） | 健康检查 |
| `/api/v1/public/b/*` | 无需鉴权 | 公开B端 |
| `/api/v1/public/c/*` | 无需鉴权 | 公开C端 |
| `/api/v1/b/*` | 需 B端 登录 | B端私有 |
| `/api/v1/c/*` | 需 C端 登录 | C端私有 |
| 其他 `/api/v1/*` | 需 B端 登录（默认匹配 `DEFAULT_B_PATTERN`） | sys、client 等 |

**关键注意**：`/api/v1/client/*` 和 `/api/v1/client-user/*` 的路径段是 `client`/`client-user`（不是 `c/`），所以匹配 `DEFAULT_B_PATTERN`，**走 B端 鉴权**，而非 C端。

**路由注册结构**（按 URL 路径分组，放在 `SetupRouters()` 中）：

```go
v1 := r.Group("/api/v1")
{
    // ── Public B（无需登录）──
    publicB := v1.Group("/public/b")
    publicB.GET("/captcha", sys_auth_captcha.GetCaptcha)
    publicB.GET("/sm2/public-key", sys_auth_sm2.GetPublicKey)
    publicB.POST("/login", sys_auth_username.Login)
    publicB.POST("/register", sys_auth_username.Register)

    // ── Public C（无需登录）──
    publicC := v1.Group("/public/c")
    publicC.GET("/captcha", client_auth_captcha.GetCaptcha)
    publicC.GET("/sm2/public-key", client_auth_sm2.GetPublicKey)
    publicC.POST("/login", client_auth_username.Login)
    publicC.POST("/register", client_auth_username.Register)

    // ── B端 私有 ──
    b := v1.Group("/b")
    b.POST("/logout", sys_auth_username.Logout)

    // ── C端 私有 ──
    c := v1.Group("/c")
    c.POST("/logout", client_auth_username.Logout)

    // ── Sys 模块（默认B端鉴权）──
    sys := v1.Group("/sys")
    sys.GET("/banner/page", ...)
    // ... 以下全部 sys 子路由

    // ── Client 模块（注意：路径是 /client，不是 /c，走默认B端鉴权）──
    client := v1.Group("/client")
    client.GET("/session/page", ...)

    // ── Client-User（注意：路径段是 client-user，走默认B端鉴权）──
    v1.GET("/client-user/page", ...)
    // ...
}
```

#### 2.14 权限系统

**`core/auth/permission_interface.go`** — `HeiPermissionInterface`
- 核心接口：运行时从数据库加载权限数据
- **`getRoleIDs(db, loginID string) []string`** — 查询 `RelUserRole` 表获取用户关联的角色 ID 列表
- **`isGroupScope(scope string) bool`** — 判断 scope 是否属于 group 维度（GROUP / GROUP_AND_BELOW / CUSTOM_GROUP）
- **`isOrgScope(scope string) bool`** — 判断 scope 是否属于 org 维度（ORG / ORG_AND_BELOW / CUSTOM_ORG）
- **`mergeDimension(cur map, scopeKey, idsKey, newScope string, newIDs []string)`** — 合并同一维度（group/org）的作用域：
  - cur 已有 SELF → 不合并（SELF 优先级最高，不可降级）
  - newScope 为 SELF → 覆盖为 SELF（最严格）
  - 否则取 `mostRestrictive(cur, new)`，CUSTOM 级别合并 ID 列表
- **`mergeScope(result map, priority string, rows []ScopeRow)`** — 合并两个路径来源的权限作用域：
  - **两条路径**：
    - P0 (DIRECT)：用户→直接授权（优先级高，值小）
    - P1 (USER_ROLE)：用户→角色→权限（优先级低）
  - **合并规则**：
    - 优先级不同 → 低 P 值（高优先级）完全覆盖
    - 优先级相同 → 按维度（group/org）分别调用 `mergeDimension` 取最严格值
- **`getAllPermissionsFromRedis() ([]string, error)`** — 从 Redis 读取 `hei:permission:keys`，遍历所有模块的权限码
- **`GetPermissionList(loginID, loginType) ([]string, error)`** — 获取用户权限码列表：
  - SUPER_ADMIN 角色 → 返回全部权限（从 Redis 读取）
  - 普通用户 → 通过 Role→Permission + User→Permission 两条路径收集
- **`GetRoleList(loginID, loginType) ([]string, error)`** — JOIN `SysRole` + `RelUserRole` 获取角色 code 列表
- **`GetPermissionScopeMap(loginID, loginType) (map[string]ScopeInfo, error)`** — 获取权限作用域映射：
  - SUPER_ADMIN → 返回所有权限的 ALL 作用域
  - 普通用户 → P1（RolePermission）+ P0（UserPermission）两条路径合并
  - 返回结构：`{"perm_code": {"group_scope": "...", "org_scope": "...", "custom_group_ids": [...], "custom_org_ids": [...]}}`
- `type ScopeInfo struct` — GroupScope, OrgScope, CustomGroupIDs, CustomOrgIDs
- `type ScopeRow struct` — PermissionCode, Scope, CustomGroupIDs, CustomOrgIDs
- 等价于 Python 版 `hei_permission_interface.py`（全部方法 + DataScope 合并逻辑）

#### 2.15 认证中间件

**`core/auth/middleware/check_login.go`** — `HeiCheckLogin(loginType ...string) gin.HandlerFunc`
- `loginType` 可选，默认 `"BUSINESS"`，传 `"CONSUMER"` 时走 C 端鉴权
- 从请求头取 token（`c.GetHeader(tokenName)`）
- 调用对应 `HeiAuthTool.IsLogin()` 或 `HeiClientAuthTool.IsLogin()`
- 未登录 → `c.Abort(); c.JSON(200, failure("未授权/未登录", 401))`
- 等价于 `@HeiCheckLogin(login_type=...)` 装饰器

**`core/auth/middleware/check_permission.go`** — `HeiCheckPermission(permissions []string, mode ...string) gin.HandlerFunc`
- 支持 `permissions` 为字符串切片（1个或多个），`mode` 可选（默认 `"AND"`）
- 先检查登录（通过 `HeiAuthTool` 或 `HeiClientAuthTool`，取决于 `login_type`）
- 单个字符串自动包装为 `[]string{permission}`（等价于 Python 的 `[permission] if isinstance(permission, str) else permission`）
- `mode == "AND"` → 调用 `HeiPermissionTool.CheckPermissionAnd(permissions...)`
- `mode == "OR"` → 调用 `HeiPermissionTool.CheckPermissionOr(permissions...)`
- 附加权限元数据到 `permissionRegistry`（通过 `RegisterPermission`，等价于 `func._hei_permission`）
- 等价于 Python 版 `@HeiCheckPermission(permission, mode, login_type)` 装饰器

**`core/auth/middleware/check_role.go`** — `HeiCheckRole(roles []string, mode ...string) gin.HandlerFunc`
- 先检查登录
- `mode == "AND"` → `HeiPermissionTool.CheckRoleAnd(roles...)`
- `mode == "OR"` → `HeiPermissionTool.CheckRoleOr(roles...)`
- 等价于 Python 版 `@HeiCheckRole(role, mode, login_type)` 装饰器

**`core/auth/middleware/client_check_login.go`** — `HeiClientCheckLogin() gin.HandlerFunc`
- 调用 `HeiCheckLogin("CONSUMER")`
- 等价于 `@HeiClientCheckLogin` 装饰器

**`core/auth/middleware/client_check_permission.go`** — `HeiClientCheckPermission(permissions []string, mode ...string) gin.HandlerFunc`

- 等价于 Python 版 `@HeiClientCheckPermission(permission, mode)` 装饰器

**`core/auth/middleware/client_check_role.go`** — `HeiClientCheckRole(roles []string, mode ...string) gin.HandlerFunc`
- 调用 `HeiCheckRole(roles, mode, CONSUMER)`
- 等价于 Python 版 `@HeiClientCheckRole(role, mode)` 装饰器

**`core/auth/middleware/norepeat.go`** — `NoRepeat(interval int) gin.HandlerFunc`
- 排除 `request`、`db`、`file` 等基础设施参数
- 剩余参数 JSON 序列化后取 hash 作为请求指纹
- 从 Redis 读取缓存指纹，检查是否在 `interval` 毫秒内有相同请求
- 已有且未超时 → `c.Abort(); c.JSON(200, failure("请求过于频繁，请X秒后再试"))`
- 无缓存或已超时 → 写入 Redis（TTL 3600s），继续执行
- 等价于 `@NoRepeat(interval=3000)` 装饰器（Python 版的 `_params_hash` 使用 `json.dumps` + `hash()`，Go 版用 `json.Marshal` + 确定性 hash）

## 四、注意事项

1. **Ent 查询**：Ent 生成的 API 以 `client.SysBanner.Query().Where(sysbanner.IDEQ(id)).First(ctx)` 方式查询，而非 SQLAlchemy 的 `select().where()`
2. **bcrypt 密码**：使用 `golang.org/x/crypto/bcrypt`，API 与 Python 的 `bcrypt` 包类似
3. **SM2 解密**：`tjfoc/gmsm` 库的 SM2 解密返回 C1C2C3 格式，需要做 C1C3C2 ↔ C1C2C3 转换（与 Python 版逻辑相同）
4. **ip2region**：使用 `lionsoul2014/ip2region/binding/golang`，API 与 Python 版略有不同但功能相同
5. **验证码图片**：使用 `golang.org/x/image/font` 和 `github.com/golang/freetype` 绘制文字，`image/draw` 绘制干扰线
