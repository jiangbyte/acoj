# Syncâ†’Async SQLAlchemy transformation spec (request path)

Apply to every `.py` under the given module dir that touches `self.db.*`, `get_db`, or calls a now-async service. The app uses SQLAlchemy 2.0 AsyncSession. `get_db` (from `sdk.infra.db`) is ALREADY an async generator yielding `AsyncSession`.

## Repository / service method bodies

AsyncSession method rules:
- `await self.db.execute(...)` â€” await it.
- `await self.db.commit()`, `await self.db.flush()`, `await self.db.refresh(x)`, `await self.db.rollback()`, `await self.db.delete(x)`, `await self.db.merge(x)` â€” await.
- `self.db.add(x)`, `self.db.add_all(xs)` â€” **stay sync, NO await**.
- Chained result calls: `.scalars()`, `.scalar_one_or_none()`, `.scalar_one()`, `.scalar()`, `.all()`, `.first()`, `.mappings()` are called on the Result returned by execute. Wrap the await:
  - `self.db.execute(stmt).scalars().all()` â†’ `(await self.db.execute(stmt)).scalars().all()`
  - `self.db.execute(stmt).scalar_one_or_none()` â†’ `(await self.db.execute(stmt)).scalar_one_or_none()`
  - `self.db.execute(stmt).scalar()` â†’ `(await self.db.execute(stmt)).scalar()`
  - multiline:
    ```
    rows = self.db.execute(
        select(...).where(...)
    ).scalars().all()
    ```
    â†’
    ```
    result = await self.db.execute(
        select(...).where(...)
    )
    rows = result.scalars().all()
    ```
    (either inline `(await ...)` wrapping OR a temp var â€” both fine; prefer temp var for multiline readability)
- Standalone execute (UPDATE/DELETE statements, no chain): `self.db.execute(sa_update(...)...)` â†’ `await self.db.execute(sa_update(...)...)`.

Every method that now contains an `await` MUST become `async def`. Propagate: if `methodA` calls `self.methodB()` and methodB became async, then `await self.methodB()` and methodA becomes async too.

Type hints: `db: Session` â†’ `db: AsyncSession`. Update imports: `from sqlalchemy.orm import Session` â†’ add `from sqlalchemy.ext.asyncio import AsyncSession` (drop `Session` if unused).

## Service getter (DI factory)
`def get_x_service(db: Session = Depends(get_db)) -> XService:` â†’
`def get_x_service(db: AsyncSession = Depends(get_db)) -> XService:`
The getter stays plain `def` (it only constructs the service with the already-resolved session). Keep `Depends(get_db)` unchanged â€” get_db is now async.

## API handlers (api/v1/api.py)
Every handler that calls a now-async service method must be `async def` and `await service.x(...)`. Handlers currently sync `def` (e.g. page/detail/remove) become `async def`. Decorator order unchanged.

## Module-level helper functions taking `db`
If a top-level helper `def _foo(db, ...)` runs `self`-less `db.execute(...)`, apply same rules â†’ `async def _foo(db, ...)` + await; update callers to `await _foo(db, ...)`.

## CPU-bound calls stay off the loop
`bcrypt.hashpw/checkpw`, image compression (`compress_base64_image`) must be wrapped in `await asyncio.to_thread(...)` once their enclosing method is async. Do NOT await `decrypt()` (SM2, fast) â€” leave inline.

## Forbidden
- Do NOT create `AsyncSessionLocal()` inside request-path code â€” use the injected `self.db`.
- Do NOT `asyncio.gather` over the same `self.db` session.
- Do NOT touch: cli/, seeds, DbLogPersister, provider.py (handled separately).
