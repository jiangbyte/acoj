# OJ系统渐进式性能测试指南

本测试套件使用k6进行OJ系统的渐进式性能测试，测试流程包括用户生成、注册、登录、代码提交和判题结果轮询。

## 测试流程

1. **生成用户数据**
2. **批量注册用户**
3. **执行性能测试**（50、100、150、200用户）
4. **生成性能报告和图表**

## 目录结构

```
_k6_test/
├── 1_generate_users.py     # 生成测试用户数据
├── 2_register_users.py     # 批量注册用户
├── performance_test.js      # k6性能测试脚本
├── generate_charts.py       # 生成性能图表和报告
├── codes/                   # 测试代码文件夹
│   └── 1985678530483163137.cpp  # 测试代码文件
└── README.md                # 测试指南
```

## 环境准备

### 安装依赖

1. **Python依赖**
   ```bash
   pip install pandas matplotlib requests
   ```

2. **k6安装**
   - 下载地址：https://k6.io/docs/getting-started/installation/
   - 安装完成后，确保k6命令可用

## 使用步骤

### 1. 生成测试用户数据

```bash
cd _k6_test
python 1_generate_users.py
```

这将生成1000个测试用户数据，保存到`test_users.csv`文件中。

### 2. 批量注册用户

```bash
python 2_register_users.py
```

- 按照提示输入并发数和请求间隔
- 注册结果将保存到`register_results_时间戳`目录中

### 3. 执行性能测试

```bash
k6 run performance_test.js
```

测试流程：
- 50用户测试（30秒预热 + 1分钟稳定 + 30秒降压）
- 休息30秒
- 100用户测试（30秒预热 + 1分钟稳定 + 30秒降压）
- 休息30秒
- 150用户测试（30秒预热 + 1分钟稳定 + 30秒降压）
- 休息30秒
- 200用户测试（30秒预热 + 1分钟稳定 + 30秒降压）

测试结果将生成：
- `performance_test_results.json` - 完整测试数据
- `performance_summary.csv` - 性能汇总数据
- `performance_summary.txt` - 文本格式的性能报告

### 4. 生成性能图表

```bash
python generate_charts.py
```

这将生成以下图表：
- `response_time_comparison.png` - 不同用户数下的响应时间对比
- `success_rate_comparison.png` - 不同用户数下的成功率对比
- `p95_response_time.png` - 不同用户数下的P95响应时间对比
- `combined_metrics.png` - 综合指标图表
- `detailed_report.txt` - 详细测试报告

## 测试结果分析

### 主要性能指标

1. **响应时间**
   - 登录时间：用户登录所需时间
   - 提交时间：代码提交所需时间
   - 判题时间：从提交到获取判题结果的总时间

2. **成功率**
   - 提交成功率：成功提交的比例
   - 会话成功率：完整会话（登录→提交→判题）成功的比例

3. **P95响应时间**
   - 95%的请求在该时间内完成

### 图表说明

1. **响应时间对比图**
   - 直观展示不同用户数下各项操作的平均响应时间
   - 便于识别性能瓶颈

2. **成功率对比图**
   - 展示不同用户数下的成功率变化
   - 反映系统在高负载下的稳定性

3. **P95响应时间图**
   - 关注系统的长尾性能
   - 更能反映真实用户体验

4. **综合指标图**
   - 同时展示响应时间和成功率
   - 便于综合分析系统性能

## 注意事项

1. **测试环境**
   - 确保测试环境与生产环境相似
   - 避免在生产环境直接进行压力测试

2. **测试数据**
   - 测试用户数据生成后可以重复使用
   - 建议每次测试前清理或备份现有测试数据

3. **测试时间**
   - 完整测试流程可能需要30分钟以上
   - 建议在系统负载较低时执行测试

4. **资源监控**
   - 建议同时监控服务器CPU、内存、磁盘和网络使用情况
   - 结合系统资源使用情况分析测试结果

## 扩展建议

1. **增加测试场景**
   - 添加不同类型的代码提交
   - 测试不同难度的题目

2. **优化测试脚本**
   - 根据实际系统调整测试参数
   - 添加更多自定义指标

3. **自动化测试流程**
   - 编写脚本自动执行完整测试流程
   - 定期执行测试并生成报告

4. **对比测试**
   - 对比不同版本系统的性能差异
   - 对比不同配置下的系统性能

## 故障排除

1. **注册失败**
   - 检查服务器地址是否正确
   - 检查验证码是否有效
   - 检查用户数据格式是否正确

2. **登录失败**
   - 确保用户已成功注册
   - 检查用户名、密码是否正确
   - 检查验证码是否有效

3. **提交失败**
   - 确保用户已成功登录
   - 检查问题ID是否存在
   - 检查代码格式是否正确

4. **k6执行错误**
   - 确保k6已正确安装
   - 检查测试脚本语法是否正确
   - 检查网络连接是否正常
